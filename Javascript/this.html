This - javascript

This es el objeto contexto de Javascript en el cual se ejecuta el código actual.

  • Todo el javascript del navegador vive dentro de "Window"
  • ¿Dónde estoy ejecutando el código?
    This cambia, dependiendo de si se está ejecutando en node o en el navegador.
  • Indicar el la dirección a seguir de un metodo en cualquier objeto.

Ejemplo: 

//This es especialmente importante en Javascript orientado a objetos.
                
const pokemon = {
    container: document.querySelector("#container"),
    renderName: function(name) {
        this.container.textContent = `Pokemon encontrado ${name}`;
    },  
    renderType: function(type) {
        this.container.textContent = `Es de tipo ${type}`;
    }
};

pokemon.renderName(Pikachu);

Bind

Bind es una función en la cual le puedes pasar el contexto deseado a this, para que en situaciones donde manejes mas de un "Objeto" en la misma aignación, puedas indicarle sobre cual trabajará el .this.
Dato importante de Bind: Si bien está asignando this a otro contexto, esto nos crea otra función. Entonces...

Ejemplo: 

const renderName = pokemon.renderName;

const who = renderName.bind(pokemon);
who("Pikachu");

Call

Si yo no quiero generar otra función y generar mi reestructuración del contexto per sé en la función; para esto existe la función Call.

Call recibe 2 parametros en la función, primero el this, y luego los parametros de la función.

Ejemplo: 

const renderName = pokemon.renderName;
renderName.call(pokemon, "Pikachu");

Implementación de .This con clases

¿Cuál es la forma mas eficiente con la que puedo trabajar en mi día a día? Respuesta: Clases.
Para optimizar esto, es mejor hacer uso de clases. 

class Pokemon {
container = document.querySelector("#container");
renderName(name) {
    this.container.textContent = `Pokemon encontrado ${name}`;
}
renderType(type) {
    this.container.textContent = `Es de tipo ${type}`;
}
}

const pikachu = new Pokemon();
pikachu.renderName("Pikachu");
pikachu.renderType("Electrico");
